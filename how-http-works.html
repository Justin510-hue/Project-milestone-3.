How HTTP Works

HTTP uses a request-and-response process. A web browser sends a request to a server, and the server sends back a response that contains a webpage, image, file, message, or other resource.

This communication happens every time a user visits a webpage, submits a form, watches a video, downloads a file, or uses an online application.

The Client

The client is usually a web browser such as Chrome, Safari, Firefox, or Edge. The browser allows the user to enter a URL, click links, submit forms, and interact with webpages.

When the user requests information, the browser creates an HTTP request and sends it across the Internet.

The Web Server

A web server stores website files and responds to requests. It may return an HTML page, an image, a stylesheet, a video, JSON data, or an error message.

Some servers also communicate with databases and application software before creating the response.

Step-by-Step HTTP Process

  1. The user enters a URL or clicks a link.
  2. The browser identifies the website’s domain name.
  3. DNS translates the domain name into an IP address.
  4. The browser connects to the web server.
  5. The browser sends an HTTP request.
  6. The server receives and processes the request.
  7. The server sends an HTTP response.
  8. The browser interprets the response and displays the page.

Example

Suppose a user enters:

https://example.com/about.html

The browser locates the server for example.com and requests the file named about.html. If the file exists, the server responds with the file and a successful status code.

If the file does not exist, the server may return a 404 Not Found response instead.

What Is in an HTTP Request?

An HTTP request usually contains several pieces of information:

  • The request method, such as GET or POST
  • The requested path or resource
  • The HTTP version
  • Headers containing browser and connection information
  • An optional body containing submitted data

A GET request normally asks for information, while a POST request usually sends information to the server.

Sample HTTP Request

GET /index.html HTTP/1.1
Host: example.com
User-Agent: Web Browser
Accept: text/html

This example asks the server at example.com to send the index.html file.

What Is in an HTTP Response?

A response usually contains:

  • The HTTP version
  • A status code
  • A short status message
  • Response headers
  • The requested content or an error message

The status code helps the browser understand whether the request was successful, redirected, rejected, or affected by a server problem.

Sample HTTP Response

HTTP/1.1 200 OK
Content-Type: text/html
Content-Length: 1250

<html>
    <body>Webpage content</body>
</html>

The status code 200 means that the request was successful.

Common HTTP Request Methods

Method Purpose Example
GET Retrieves information from a server. Opening a webpage
POST Sends new information to a server. Submitting a registration form
PUT Replaces or updates a resource. Updating an account record
PATCH Updates part of an existing resource. Changing a profile picture
DELETE Requests removal of a resource. Deleting a saved item
HEAD Requests headers without the response body. Checking whether a file exists

HTTP Status Code Categories

Category Meaning Example
100–199 Informational responses 100 Continue
200–299 Successful responses 200 OK
300–399 Redirection responses 301 Moved Permanently
400–499 Client error responses 404 Not Found
500–599 Server error responses 500 Internal Server Error

Common Status Codes

  • 200 OK: The request was successful.
  • 201 Created: A new resource was created.
  • 301 Moved Permanently: The resource moved to a different address.
  • 400 Bad Request: The request was invalid.
  • 401 Unauthorized: Authentication is required.
  • 403 Forbidden: The server refuses access.
  • 404 Not Found: The resource could not be located.
  • 500 Internal Server Error: The server experienced a problem.

How HTTPS Protects Communication

HTTPS combines HTTP with TLS encryption. Before sensitive information is transferred, the browser and server create a secure connection.

Encryption helps prevent attackers from reading passwords, payment information, private messages, and account data while it travels between the client and server.

How Browsers Improve Performance

Browsers use caching to save copies of files that have already been downloaded. When the user returns to a website, some resources can be loaded from the local device instead of being downloaded again.

Modern versions of HTTP also allow multiple resources to be transferred efficiently through a smaller number of connections.
